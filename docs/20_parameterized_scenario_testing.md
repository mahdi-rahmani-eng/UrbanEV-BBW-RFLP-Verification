# 20 - Parameterized Scenario Testing

## Purpose

This document explains the parameterized scenario testing extension added to the project.

The purpose is to move beyond a small set of manually defined scenarios and create an ODD-constrained scenario space for broader MATLAB-based verification.

This extension strengthens the project while keeping it suitable for a university-level concept-design and verification workflow.

---

## Why Parameterized Scenario Testing Was Added

The original project includes six manually defined scenarios.

Those scenarios are useful for basic requirement verification, but they cover only a small number of cases.

Parameterized scenario testing improves the project by varying key input parameters such as:

- ego speed;
- road condition;
- road friction;
- actuator delay;
- target distance;
- relative speed;
- scenario family;
- target type.

This allows the warning and brake-request logic to be checked over a broader set of dry-road, wet-road, and delay-sensitive cases.

---

## Scenario Generation Script

The parameterized scenarios are generated by:

`matlab/generate_parameterized_scenarios.m`

This script creates:

`data/parameterized_scenarios.csv`

and:

`figures/scenario_parameter_space.png`

---

## Scenario Parameters

| Parameter | Values / Meaning |
|---|---|
| ego_speed_kmh | 10, 20, 30, 40, 50 km/h |
| road_condition | dry or wet |
| friction_mu | dry = 0.75, wet = 0.45 |
| actuator_delay_s | 0.10, 0.20, 0.30 s |
| scenario_family | NORMAL_FOLLOWING, CLOSE_TARGET, HIGH_CLOSING_RISK, SUDDEN_OBSTACLE |
| target_type | lead_vehicle or stationary_obstacle |
| expected_state | SAFE, CAUTION, WARNING, or EMERGENCY_BRAKE |

---

## Scenario Families

| Scenario Family | Intended Behaviour | Typical Expected State |
|---|---|---|
| NORMAL_FOLLOWING | Safe distance and low closing risk | SAFE |
| CLOSE_TARGET | Close but not critical target | CAUTION |
| HIGH_CLOSING_RISK | Higher risk due to distance and relative speed | WARNING |
| SUDDEN_OBSTACLE | Critical obstacle case | EMERGENCY_BRAKE |

---

## Batch Verification Script

The generated parameterized scenarios are verified by:

`matlab/scenario_batch_verification.m`

This script reads:

`data/parameterized_scenarios.csv`

and generates:

`results/parameterized_scenario_results.csv`

It also generates:

- `figures/parameterized_verification_summary.png`
- `figures/parameterized_warning_state_distribution.png`

---

## Verification Logic

For each parameterized scenario, the script calculates:

- stopping distance;
- TTC;
- calculated warning state;
- brake request;
- pass/fail result.

The calculated warning state is compared with the expected state defined during scenario generation.

A scenario passes if:

`calculated_state == expected_state`

---

## Main Outputs

| Output File | Purpose |
|---|---|
| data/parameterized_scenarios.csv | Stores the generated parameterized scenario set. |
| results/parameterized_scenario_results.csv | Stores calculated outputs and pass/fail results. |
| figures/scenario_parameter_space.png | Shows the generated scenario parameter space. |
| figures/parameterized_verification_summary.png | Shows pass/fail count for all generated scenarios. |
| figures/parameterized_warning_state_distribution.png | Shows distribution of calculated warning states. |

---

## Interpretation of Results

The current parameterized scenario set contains 120 generated scenarios.

The batch verification checks whether the warning logic behaves consistently across the generated ODD-constrained scenario space.

The result strengthens the project because verification is no longer limited to only six manually defined scenarios.

Instead, the project now includes a broader parameterized test set covering:

- multiple speed levels;
- dry and wet road cases;
- multiple actuator-delay values;
- multiple risk families;
- lead-vehicle and stationary-obstacle target interpretations.

---

## Link to ODD

The generated parameterized scenarios remain inside the defined Operational Design Domain.

They are limited to:

- low-speed urban/campus operation;
- ego speed up to 50 km/h;
- dry or wet asphalt;
- straight-road assumption;
- clear visibility assumption;
- simplified scenario-level input;
- lead vehicle or stationary obstacle.

Out-of-ODD conditions such as pedestrians, cyclists, snow, ice, fog, curves, intersections, highway operation, and real perception are not included.

---

## Scope Note

This is a lightweight parameterized scenario-testing extension.

It is inspired by modern scenario-based verification practices but does not claim ISO 34502 compliance, ASAM OpenSCENARIO implementation, ASAM OpenODD implementation, SOTIF compliance, or production AEB validation.

---

## Conclusion

Parameterized scenario testing improves the project by adding broader, repeatable, ODD-constrained verification.

It makes the MATLAB verification stronger and more suitable for automotive systems engineering and ADAS-oriented internship discussion while keeping the project realistic and university-level.
